1なぜ PCA では不十分か

標準的なPCAは「分散を最大化する軸」を見つけます。問題は、その軸が複数のタスク変数を混ぜてしまうことです。

図1. PCAはタスク変数を混合してしまう（左）。dPCAはANOVA分解で各因子に対応するsubspaceを分離する（右）。

2核心アイデア

一言で： X を「rule が原因の部分」「stim が原因の部分」... に分解（ANOVA）し、それぞれに PCA をかける。 
X = X_t + X_r + X_s + X_tr + X_ts + X_rs + X_trs   （ANOVA分解）

各 X_φ に SVD をかける：
  SVD(X_φ) → D_φ（demixed decoder）

射影：
  Z_r = D_rᵀ X  ← 「ruleに関する表現」だけを抽出
  Z_s = D_sᵀ X  ← 「stimに関する表現」だけを抽出

3基礎：2要因ANOVAとの対応

dPCAのANOVA分解は統計学の2要因ANOVAと全く同じ考え方です。

図2. 統計学のANOVAとdPCAのANOVA分解の対応。dPCAでは「ニューロン」がANOVAの「繰り返し」に対応する。

4マージナライゼーション（ANOVA分解）の計算

これがdPCAで最も重要かつ最も難しい部分です。小さいサブセットから順番に計算するのが鍵です。

具体例：labels = "ts"（time と stim の2要因）

データ：X[N, T, S]（N=ニューロン、T=time、S=stim）

図3. 2要因（time × stim）でのANOVA分解の手順。小さいサブセットから順番に計算し、各ステップで下位効果を引いて純化する。

3要因の場合（labels = "trs"）

サブセットは2³−1 = 7個。計算は必ず小さいサブセットから順番に行います。

サブセット（サイズ順）：
  size 1: t, r, s
  size 2: tr, ts, rs
  size 3: trs

計算：
  X_t   = mean_{r,s}(X) - μ
  X_r   = mean_{t,s}(X) - μ
  X_s   = mean_{t,r}(X) - μ
  X_tr  = mean_s(X) - X_t - X_r - μ
  X_ts  = mean_r(X) - X_t - X_s - μ
  X_rs  = mean_t(X) - X_r - X_s - μ
  X_trs = X - X_t - X_r - X_s - X_tr - X_ts - X_rs - μ
包除原理（Möbius inversion）： X_φ を計算するとき、「φ のすべての strict subset ψ ⊊ φ」に対応する X_ψ を引く。 これを省くと下位の主効果が混入してしまい、直交性が崩れます。

5Decoder の計算（SVD）

各 X_φ を [N × K] に unfold（Kは条件数の積）してから SVD をかけます。

SVD( X_φ [N × K] ) = U Σ Vᵀ

D_φ = U[:, 1:k_φ]   ← top k_φ 左特異ベクトル

var_exp_φ = Σ[1:k_φ]² / sum(Σ²)  ← 説明分散比

図4. 全データXにSVDをかけると混合成分（左）、マージナライズされたX_ruleにSVDをかけるとruleだけの成分（右）が得られる。

6Encoder の計算（双直交性）

Decoder D_φ はデータを低次元に射影します。Encoder F_φ は低次元から元の空間に「戻す」ための行列で、標準PCAとは違い D_φ ≠ F_φ です。

射影

Z_φ = D_φᵀ X_2d   [k_φ × K]

注意：X_φ ではなく X（全データ）を射影する
← D_φ は X_φ の方向だが、全データに適用してこそ意味がある

Encoderの導出（最小二乗）

X_φ ≈ F_φ Z_φ

→ F_φ = X_φ Zᵀ_φ (Z_φ Zᵀ_φ)⁻¹   [N × k_φ]

図5. 標準PCAではencoder=decoder（左）だが、dPCAではbiorthogonalityの条件から両者が分かれる（右）。これにより各latent次元が1つの因子だけに対応することが保証される。

7アルゴリズム全体

1
ANOVA分解（dpca_marginalize）
全体平均 μ[n] = mean over all conditions

for each subset φ in powersets(labels), small first:
  avg_dims = dimensions NOT in φ
  X_φ_raw  = mean over avg_dims  → broadcast to shape of X
  X_φ      = X_φ_raw - μ - sum(X_ψ for all ψ ⊊ φ)
  X_φ_2d   = unfold X_φ to [N × K]
検証：sum(X_φ) ≈ X_2d - μ（Frobenius誤差 < 1e-10）、かつ各 X_φ ペアの Frobenius 内積 ≈ 0
2
Decoder 計算（SVD）
for each φ:
  g(U, s, V) %=% svd(X_φ_2d)    ← R: svd()はd=s, u=U, v=V
  D_φ = U[, 1:k_φ]
  var_exp_φ = cumsum(s^2) / sum(s^2)
検証：D_φの各列は単位ベクトルかつ直交（t(D_φ) %*% D_φ ≈ I）
3
Encoder 計算
for each φ:
  Z_φ  = t(D_φ) %*% X_2d          ← 全データを射影
  F_φ  = X_φ_2d %*% t(Z_φ) %*%
         solve(Z_φ %*% t(Z_φ) + α * diag(k_φ))
検証（biorthogonality）：t(D_φ) %*% F_φ ≈ diag(k_φ)（対角行列）
4
射影（Transform）
for each φ:
  Z_φ = t(D_φ) %*% X_2d          ← [k_φ × K]
  reshape Z_φ to [k_φ × T × n_r × n_s × ...]
Output： D_φ（decoders）、F_φ（encoders）、Z_φ（projected data per factor）、var_exp（explained variance）

8dPCA → jPCA への接続

dPCA で得られた Z_r [k_r × T × n_rule] を
list of condition matrices に変換：

X_list_r = list of [k_r × T] matrices, one per rule condition

jpca_fit(X_list_r)  → rotation in rule subspace
なぜこの組み合わせが有効か：
raw EEGに直接jPCAをかけると、rotationがrule由来かstim由来か不明。
dPCAでrule subspaceを分離した後にjPCAをかけることで、 「ruleに関する表現だけが回転するか」を問うことができる。

比較として：

R²_ratio of jpca_fit(Z_r)   ← rule subspace のrotation
R²_ratio of jpca_fit(Z_s)   ← stim subspace のrotation
R²_ratio of jpca_fit(Z_rs)  ← rule×stim交互作用 のrotation

「rule subspace だけが回転する」なら conjunctive representation の証拠になりうる

9Comprehension Check

  1. なぜ X_ts の計算で X_t と X_s を引くのか？ ヒント：X_ts = mean_r(X) だけだと X_ts に time の主効果が含まれてしまう。引き算することで何が残るか？
  2. X_t と X_s が Frobenius 直交することを、toy data（N=2, T=3, S=2）で手計算で確かめよ。 ヒント：trace(X_tᵀ X_s) を計算する。X_t は S次元に定数、X_s は T次元に定数なので積が打ち消し合う。
  3. なぜ D_φ の計算で X_φ を使い、Z_φ の計算では X_2d（全データ）を使うのか？ ヒント：D_φ は「X_φ の分散を最大化する軸」。全データに適用することで「rule方向の表現の時間変化」全体が見える。
  4. biorthogonality D_φᵀ F_φ ≈ I が崩れると何が起きるか？ ヒント：D_φ = F_φ（PCA的な使い方）にした場合、Z_φ = D_φᵀ X に含まれる分散は何に由来するか？
  5. N < K のとき regularizer が不要な理由は？ ヒント：Z_φ Z_φᵀ は [k_φ × k_φ] の小さな行列。N < K なら k_φ << K なので通常可逆。
  6. dPCA → jPCA で、jPCAを Z_r ではなく X_raw に直接かけるとどんな問題があるか？ ヒント：raw EEGでrotationが検出されたとき、それがrule/stim/time/ノイズのどれによるものか区別できない。
